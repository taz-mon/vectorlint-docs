## Before you begin

VectorLint requires Node.js. Use an LTS release for best compatibility.

You will also need an API key for a supported LLM provider — OpenAI, Anthropic, Gemini, or Azure. VectorLint sends your content to the provider you configure for evaluation.

---

## Step 1: Initialize your project

Run the quick-init command in your project root:

```bash
npx vectorlint init --quick
```

This creates a `VECTORLINT.md` file in your project root. VectorLint automatically generates a "Style Guide Compliance" rule from its contents and uses it for all evaluations.

Open `VECTORLINT.md` and paste in your style guide — or add a few plain-language instructions to start:

```md
- Use active voice.
- Avoid jargon and unexplained acronyms.
- Write in second person (you, your).
- Keep sentences under 25 words where possible.
```

<Tip>
  You don't need a comprehensive style guide to get useful output. A few clear instructions are enough to run a meaningful first check.
</Tip>

---

## Step 2: Configure your LLM provider

<Warning>
  You must complete this step before running a check. VectorLint will not produce output without a configured provider and valid API key.
</Warning>

Open `~/.vectorlint/config.toml` and set your provider and API key.

For OpenAI:

```toml
[env]
LLM_PROVIDER = "openai"
OPENAI_API_KEY = "sk-..."
```

For Anthropic:

```toml
[env]
LLM_PROVIDER = "anthropic"
ANTHROPIC_API_KEY = "sk-ant-..."
```

<Tip>
  You can also set these values in a `.env` file in your project root. Project-level values take precedence over `config.toml`.
</Tip>

---

## Step 3: Run a check

Point VectorLint at any Markdown file in your project:

```bash
npx vectorlint path/to/doc.md
```

VectorLint evaluates the file against your `VECTORLINT.md` instructions and returns findings in your terminal.
